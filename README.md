## Project Background
- ### 필요성
  - 기존의 MAC 주소, IMEI 기반 디바이스 인증 방식은 스푸핑 공격 및 분실·도난 기기에 취약함.
  - IoT 환경 확산으로 수많은 기기 접근을 안전하게 통제할 수 있는 체계가 요구됨.
  - 네트워크 경계가 사라진 환경에서 “모든 요청을 신뢰하지 않는다”는 제로 트러스트 보안 모델의 적용 필요성이 대두됨.
- ### 기존 해결책의 문제점
  - 단일 식별자 기반의 인증은 위조나 복제가 용이함.
  - 인증이 최초 접속 시점에만 이루어져 지속적인 검증(Continuous Verification) 이 불가능함.
  
## System Design
  - ### System Requirements
    - 클라이언트: 기기 식별자(MAC) 자동 전송 및 OTP 인증 UI 제공
    - 서버: 등록된 기기 검증, 미등록 기기 승인 절차, OTP 생성·검증, 지속 검증 수행
    - 정책 엔진: 기기 상태(운영체제 버전, 보안 패치 등) 기반 접근 정책 자동 적용
    
## Case Study
  - ### Description
    - Python 기반 서버-클라이언트 구조 구현 (PyQt5 GUI)
    - pyotp를 이용한 OTP 인증, socket 통신 기반 연결
    - 등록되지 않은 기기 접근 시 서버 관리자 승인 팝업 작동
    - 승인 시 MAC 주소가 device_list.txt에 자동 저장
    - 15초 유효 OTP를 통한 실시간 인증 및 세션 무결성 유지
  
  
## Conclusion
  - ### 본 연구는 제로 트러스트 보안 원칙(지속 검증·최소 권한·동적 정책 적용) 을 디바이스 인증에 도입함으로써, 기존 방식의 취약점을 개선하였다.
  - ### 관리자 승인 및 OTP 기반 다중 인증 구조로 스푸핑·분실·도난 기기 접근을 차단하였다.
